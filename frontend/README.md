# Frontend

Next.js/TypeScript control panel for running Alfred workflows locally.

## Contents

- `.next/` – build artifacts and server-side rendering cache generated by `npm run dev`/`npm run build`.
- `app/` – Next.js 13+ App Router source (pages, components, styles).
- `components/` – shared React components used across the UI.
- `node_modules/` – installed npm dependencies (managed via `npm install`).
- `pages/` – legacy Pages Router endpoints (if any remain).
- `.eslintrc.json` – linting rules for the frontend codebase.
- `next-env.d.ts` – automatically generated TypeScript definitions for Next.js globals.
- `next.config.js` – Next.js configuration (rewrites, env exposure, etc.).
- `package.json` / `package-lock.json` – dependency manifest and lockfile for reproducible installs.
- `tsconfig.json` – TypeScript compiler configuration shared across the app.
